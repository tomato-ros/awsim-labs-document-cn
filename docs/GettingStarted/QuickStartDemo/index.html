    快速入门演示#

    您可以在下面找到有关如何设置由 Autoware 控制的 AWSIM 模拟的自动驾驶演示的说明。 该说明假定使用 Ubuntu 操作系统.

    Demo 配置#

    AWSIM 演示中提供的模拟配置如下:

    AWSIM Demo Settings
    Vehicle Lexus RX 450h
    Environment Japan Tokyo Nishishinjuku
    Sensors GNSS
    IMU
    3 x VLP16
    Traffic Light Camera
    Traffic Randomized traffic
    ROS2 humble

    PC 规格#

    请确保您的机器满足以下要求，以便正确运行模拟:

    Required PC Specs
    OS Ubuntu 22.04
    CPU 6c12t or higher
    GPU RTX 2080 or higher
    Nvidia Driver (Ubuntu 22) 545

    DDS 配置#

    为了以最佳性能运行 AWSIM Labs 且不占用网络，请按照以下步骤作.

    将以下行添加到 ~/.bashrc 文件:

    if [ ! -e /tmp/cycloneDDS_configured ]; then
    sudo sysctl -w net.core.rmem_max=2147483647
    sudo sysctl -w net.ipv4.ipfrag_time=3
    sudo sysctl -w net.ipv4.ipfrag_high_thresh=134217728     # (128 MB)
    sudo ip link set lo multicast on
    touch /tmp/cycloneDDS_configured
fi

    每次重新启动此计算机并打开新终端时，都会执行上述命令。

    在重新启动计算机之前，它们不会再次执行.

    CycloneDDS 配置#

    将以下内容另存为 cyclonedds.xml 在您的主目录中 ~: 

    <?xml version="1.0" encoding="UTF-8" ?>
<CycloneDDS xmlns="https://cdds.io/config" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="https://cdds.io/config https://raw.githubusercontent.com/eclipse-cyclonedds/cyclonedds/master/etc/cyclonedds.xsd">
    <Domain Id="any">
        <General>
            <Interfaces>
                <NetworkInterface name="lo" priority="default" multicast="default" />
            </Interfaces>
            <AllowMulticast>default</AllowMulticast>
            <MaxMessageSize>65500B</MaxMessageSize>
        </General>
        <Internal>
            <SocketReceiveBufferSize min="10MB"/>
            <Watermarks>
                <WhcHigh>500kB</WhcHigh>
            </Watermarks>
        </Internal>
    </Domain>
</CycloneDDS>

    确保将以下行添加到 ~/.bashrc 文件:

    export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
export CYCLONEDDS_URI=/home/your_username/cyclonedds.xml

    取代 your_username 替换为您的实际用户名.

    注意

    您应该使用指向 cyclonedds.xml 文件.

    警告

    需要重新启动系统才能使这些更改正常工作.

    警告

    不要 设置 export ROS_LOCALHOST_ONLY=1. CycloneDDS 配置就足够了.

    运行 AWSIM 演示#

    要运行模拟器，请按照以下步骤作.

    1. 安装 Nvidia GPU driver (如果已安装，则跳过).

      1. 添加 Nvidia driver 到 APT 存储库 
        sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt update
      2. 安装推荐的驱动程序版本. 
        sudo ubuntu-drivers autoinstall

# or install a specific version (following was tested)
sudo apt install nvidia-driver-545
      3. 重新启动计算机，使系统检测到已安装的驱动程序. 
        sudo reboot
      4. 打开终端并检查 nvidia-smi 命令是否可用，并输出类似于下面显示的摘要. 
        $ nvidia-smi
+---------------------------------------------------------------------------------------+

|-----------------------------------------+----------------------+----------------------+
| GPU  Name                 Persistence-M | Bus-Id        Disp.A | Volatile Uncorr. ECC |
| Fan  Temp   Perf          Pwr:Usage/Cap |         Memory-Usage | GPU-Util  Compute M. |

|                                         |                      |                  N/A |
+-----------------------------------------+----------------------+----------------------+
...

    2. 安装 Vulkan Graphics Library (如果已安装，则跳过).

      1. 更新环境. 
        sudo apt update
      2. 安装库. 
        sudo apt install libvulkan1

    3. 下载并运行 AWSIM 演示二进制文件.

      1. 从以下位置下载最新版本:

        AWSIM Labs GitHub Releases Page

      2. 解压缩下载的文件.

      3. 使文件可执行.

        右键单击 awsim_labs.x86_64 文件并检查 Execute 复选框

        或执行以下命令.

        chmod +x <path to AWSIM folder>/awsim_labs.x86_64

      4. 启动 awsim_labs.x86_64. 

        ./<path to AWSIM folder>/awsim_labs.x86_64

    启动 Autoware#

    要使用 AWSIM 演示配置和运行 Autoware 软件，请：

    1. 下载 map files (pcd, osm) 并解压缩它们. Download Map files (pcd, osm)

    2. 克隆 Autoware 并移动到目录. 

      git clone https://github.com/autowarefoundation/autoware.git
cd autoware
    3. 将 branch 切换到 main. 
      git checkout main
    4. 配置环境. (如果之前已配置 Autoware 环境，则跳过) 
      ./setup-dev-env.sh
    5. 创建 src 目录并将外部依赖存储库克隆到其中. 
      mkdir src
vcs import src < autoware.repos
    6. 安装依赖的 ROS 软件包. 
      source /opt/ros/humble/setup.bash
rosdep update
rosdep install -y --from-paths src --ignore-src --rosdistro $ROS_DISTRO

    7. 构建工作区. 

      colcon build --symlink-install --cmake-args -DCMAKE_BUILD_TYPE=RelWithDebInfo -DCMAKE_EXPORT_COMPILE_COMMANDS=1

    8. 启动 Autoware. 

      source install/setup.bash
ros2 launch autoware_launch e2e_simulator.launch.xml vehicle_model:=awsim_labs_vehicle sensor_model:=awsim_labs_sensor_kit map_path:=<absolute path of map folder> launch_vehicle_interface:=true

# 使用 map 文件夹的绝对路径，不要使用 ~ 运算符.

# 例:
ros2 launch autoware_launch e2e_simulator.launch.xml vehicle_model:=awsim_labs_vehicle sensor_model:=awsim_labs_sensor_kit map_path:=/home/your_username/autoware_map/nishishinjuku_autoware_map launch_vehicle_interface:=true
