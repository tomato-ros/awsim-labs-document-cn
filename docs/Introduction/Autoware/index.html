    Autoware#

    Autoware 是专为自动驾驶应用设计的开源软件平台。它的创建旨在为开发和测试自动驾驶汽车系统提供一个全面的框架。自动软件 提供了一系列模块和库，可协助完成与感知、规划和控制相关的各种任务，使研究人员和开发人员能够更轻松地构建自动驾驶系统.

    Autoware 的主要目的是通过提供强大而灵活的平台来支持自动驾驶技术的发展。它旨在通过提供即用型软件堆栈来加速自动驾驶汽车的研究和部署。自动软件 专注于城市驾驶场景，支持 LiDAR、Radar、Cameras 等各种传感器，允许感知车辆的周围环境.

    为什么将 AWSIM 与 Autoware 结合使用?#

    Autoware 可以与 AWSIM 一起使用，原因有几个。首先，像 AWSIM 这样的模拟器提供了一个经济高效且安全的环境，用于在将自动驾驶算法部署到真实车辆之前对其进行测试和验证。Autoware 与模拟器的集成使开发人员能够评估和微调他们的算法，而不会有实际事故或损坏的风险.

    此外，仿真器使开发人员能够重现复杂的驾驶场景，包括困难条件或罕见事件，这些场景可能难以在如此高保真度的实际测试中复制。Autoware 与 AWSIM 的兼容性允许软件和模拟车辆之间的无缝集成，从而实现对自动驾驶功能的全面测试和验证。通过使用模拟器，Autoware 可以在各种场景中进行广泛的测试，以确保其稳健性和可靠性.

    架构#

    在架构方面，Autoware 遵循模块化方法。它由多个独立的模块组成，这些模块通过 ROS2 相互通信。这种模块化结构允许用户根据他们的特定需求和要求选择和组合不同的模块。软件堆栈由多个组件组成，包括感知、定位、规划和控制模块。以下是每个模块的简要概述:

    • 传感 - 从安装在自动驾驶汽车上的不同传感器(如 LiDAR、GNSS、IMU 和 摄像头)获取数据。它对接收到的数据进行预处理，以便稍后通过 感知 模块提取有关周围环境的相关信息，或通过 定位 模块提取有关车辆位置的相关信息。 更多详情 查看这里.
    • 感知 - 对传感器数据(LiDAR、摄像头)进行高级处理，以提取有关周围环境的有意义信息。它执行对象检测(其他车辆、行人)、车道检测和交通信号灯识别等任务。更多详情 点击此处.
    • 定位 - 对来自 传感 模块(如 LiDAR、GNSS、IMU)和里程计传感器的数据进行融合，以准确估计车辆的位置和方向。更多详情 查看这里.
    • 规划 - 根据从 Perception 和 Localization 收集的信息为自动驾驶汽车生成安全可行的轨迹。它还考虑了 Map 中的各种因素，例如交通规则和路况。更多详情 查看这里.
    • 控制 - 通过向车辆的执行器发送命令(例如转向、油门和制动)来执行计划的轨迹。它确保车辆在保持安全性和稳定性的同时遵循所需的轨迹。更多详情 查看这里.
    • 车辆接口 - 是实现 Autoware 软件系统和车辆之间通信和交互的关键组件。它促进了自动驾驶作所需的控制信号和车辆信息的交换。车辆接口确保 Autoware 可以向车辆发送命令，例如加速、制动和转向，同时还可以接收来自车辆的实时数据，例如速度、位置和传感器数据。它就像一座桥梁，让Autoware能够无缝地与它所驾驶的车辆的特定特性和要求对接。更多详情 查看这里.
    • 地图 - 地图模块创建并维护自动驾驶汽车运行环境的表示。它结合了来自 Lanelet2 ('.osm') 和 PointCloud ('.pcd') 的数据来生成详细的地图。该地图包含有关道路几何形状、车道标记、交通信号灯、规则和其他相关要素的信息。地图是规划和决策过程的重要参考。更多详情 查看这里.
