    与 Autoware 和 Scenario simulator v2 相结合#

    Scenario Simulator v2 (SS2) 是专为 Autoware, 一个开源的自动驾驶软件平台.它作为 Autoware 开发人员的工具，可以方便地在不同模拟器上创建和执行场景.

    SS2 的主要目标是为 Autoware 开发人员提供一种有效的方法，可以一次性编写场景，然后在多个模拟器中执行它们.通过提供对不同模拟器和场景描述格式的支持，该框架确保了灵活性和兼容性.

    此框架中的默认方案格式为 TIER IV Scenario Format version 2.0.在此格式上定义的场景由 'scenario_test_runner' 转换为 'openSCENARIO' 格式，然后由 'openscenario_interpreter' 解释.基于这种解释，'traffic_simulator' 模拟了城市地区的交通流.每个 NPC 都有一个行为树，并执行场景中的命令.

    该框架使用 ZeroMQ 进程间通信来实现模拟器和 traffic_simulator 之间的无缝交互.为了确保模拟器的同步运行，SS2 利用了 ZeroMQ 提供的 Request/Reply 套接字，并通过 协议缓冲区 交换二进制数据.这使模拟器能够以同步方式运行，从而提高场景测试的准确性和可靠性.

    组合架构#

    AWSIM 与 [默认场景](../AWSIM/ 的包. 这里 SS2 中的 traffic_simulator 取代了 AWSIM 中的 TrafficSimulator 实现 - 因此，它及其 StopLines、TrafficLanes 和 TrafficIntersection 组件被删除.此外，NPCPedestrian 和 NPCVehicles 没有被添加为 Environment.

    相反，它们的对应项被添加到负责生成场景的 Entities 的 ScenarioSimulatorConnector 对象中. Entities 可以是： Pedestrian, Vehicle, MiscObject 和 Ego. EgoEntity 等同于 EgoVehicle ，它也已从默认场景中删除.然而，它具有相同的组件 - 它仍然与 Autoware 通信，如 这里 所述. 所以可以认为 EgoVehicle 没有改变，NPCPedestrians 和 NPCVehicles 现在由 SS2 直接控制.

    SS2 架构的详细说明可在 此处 找到. 可以找到 SS2 和 Autoware 之间通过 ROS2 进行通信的描述 这里.

    序列图#

    在序列图中，负责 AWSIM 与 Autoware 通信的部分也保持不变. 描述 这里 是下图所示的引用的有效说明.

    SS2 和 AWSIM 之间的通信通过 Request-Response 消息进行，如下所示:

    1. Launch - Autoware 已启动并初始化.
    2. Initialize - 初始化 AWSIM 中的环境，设置基本参数.
    3. opt Ego spawn - 可选，则 EgoEntity (带有传感器) 在场景中定义的配置中生成.
    4. opt NPC spawn loop - 可选，则场景中定义的所有 Entities (NPC) 都会生成，场景可以包含任意数量的每种 Entity 类型，也可能根本不包含它们，或者也可以是可用实体的任意组合.
    5. update loop - 这是执行场景命令的主循环，首先更新 EgoEntity - SS2 获取其状态，然后所有其他 Entity 更新-每个 NPC 的状态根据场景设置.接下来，更新模拟帧 - 这里发生 Autoware 和 AWSIM 之间的通信.循环的最后一步是更新红绿灯状态.
    6. despawn loop - 场景结束后，场景中生成的所有 Entities 都会消失(包括 EgoEnity)
    7. Terminate - Autoware 已终止.
